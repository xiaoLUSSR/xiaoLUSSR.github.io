安裝說明
如果您過去克隆過我的儲存庫並希望確保您擁有最新更新，請從命令行上的 SteamDeck_rEFInd 資料夾中執行 。如果發現缺少任何檔案或任何檔案需要更新，則可以刪除目錄並重新克隆，也可以執行 。您可以稍後與另一個 ，如果沒有任何遺漏或更改，它應該顯示 “最新”。git statusgit reset –hard origin/maingit status

在桌面模式下從 SteamOS 命令行中，逐個運行這些命令。

git clone https://github.com/jlobue10/SteamDeck_rEFInd/
cd SteamDeck_rEFInd
chmod +x SteamDeck_rEFInd_install.sh
./SteamDeck_rEFInd_install.sh

如果您嘗試過 rEFInd 並決定不想再使用它，則可以使用此命令禁用 rEFInd EFI 條目（將 XXXX 替換為 rEFInd EFI 條目號）。

sudo efibootmgr -b XXXX -A

如果要刪除 rEFInd EFI 條目，請運行以下命令。

sudo efibootmgr -b XXXX -B

卸載已安裝的 rEFInd 包附帶的包和檔，請運行。pacman

sudo steamos-readonly disable

These next two commands may not be necessary, but they don’t hurt anything either

sudo pacman-key –init
sudo pacman-key –populate archlinux

The following command performs the pacman refind package removal

yes | sudo pacman -R refind
sudo steamos-readonly enable
非 pacman 安裝文本文件刪除起來會稍微複雜一些，但如果有人堅持這樣做，不會太難。由於這些檔只佔用 5GB 根分區上的非常小的空間（不佔用分區上遊戲的任何空間），因此我不會在這裡更詳細地討論它。/home

要從分區中刪除 rEFInd 目錄（請注意，在此處出錯並刪除 /esp 分區上的錯誤檔或資料夾可能會導致您的 Steam Deck 無法啟動並需要恢復映像。認為這是一個公平的警告，我對這裡的使用者錯誤不承擔任何責任。運行此命令。/esp

sudo rm -rf /esp/efi/refind/
